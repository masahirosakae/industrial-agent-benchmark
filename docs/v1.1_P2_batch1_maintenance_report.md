## 7. generic_answer_penalty 設定状況

全 10 問で以下の標準条件を `enabled: true` で設定済み。
非標準 condition は使用していないため、validator warning は 0。

```yaml
generic_answer_penalty:
  enabled: true
  conditions:
    - scenario_specific_numbers_not_used
    - no_priority_order
    - no_tradeoff_analysis
    - generic_framework_only
    - no_named_stakeholder
```

狙い:
- 数値を使わない一般論回答を抑制
- 優先順位のない保全計画を減点
- 安全 / 品質 / 納期 / コストのトレードオフ欠落を減点
- 「PDCA」「予知保全導入」だけの汎用回答を減点
- QA / 安全責任者 / 保全責任者など承認者不在の回答を減点

## 8. structured_output_requirements 設定状況

全 10 問で JSON 出力を必須化した。

```yaml
structured_output_requirements:
  required: true
  format: json
  required_fields:
    - decision
    - rationale
    - numeric_assessment
    - risks
    - required_approvals
    - next_actions
```

`decision` / `rationale` / `risks` / `required_approvals` / `next_actions` に加え、Maintenance & Engineering では
数値判断を強制するため `numeric_assessment` を必須フィールドに含めた。

## 9. disallowed_answers / critical_failures 適用状況

全 10 問で以下を満たしている。

| 要件 | 実績 |
|---|---|
| `disallowed_answers` 3 件以上 | 全問 3 件 |
| `critical_failures` 3 件以上 | 全問 3 件 |

重点的に禁止した回答類型:
- 安全インターロック / LOTO / proof test の軽視
- データ不完全 (TAN 記録不備、振動とトルクの矛盾) の無視
- 品質確認なしの量産復帰・通常出荷
- ROI / NPV / Availability / 期待損失を計算しない一般論回答
- サンクコスト誤謬による投資判断
