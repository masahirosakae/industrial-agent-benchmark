# Industrial Agent Benchmark v1.1-pre Release Notes

## 1. リリース概要

Industrial Agent Benchmark v1.1-pre は、v1.0 で凍結した 90 問のベンチマークを基盤としつつ、製造業向け Agent の能力差を **より明確に検出できる評価設計** へ拡張したプレリリースである。

v1.0 では「上位モデル間で final_score が 4〜5 点に集中し、モデル差が出にくい」点が課題だった。
v1.1 では、シナリオ固有の数値条件、`score_cap_rules`、`numeric_checks`、`generic_answer_penalty`、`structured_output_requirements`、`disallowed_answers`、強化された `critical_failures`、Agent 層の権限境界評価などを導入し、識別力を改善した。

本リリースは正式版 (v1.1.0) の前段として、ベンチマーク本体、Judge v2、評価サブセット、および識別力検証結果を公開するためのものである。

## 2. v1.0 → v1.1 変更点

### 2.1 ベンチマーク本体

| 項目 | v1.0 | v1.1-pre |
|---|---:|---:|
| 総問題数 | 90 | **140** |
| Layer 1 (Industrial Knowledge) | 40 | 50 |
| Layer 2 (Industrial Reasoning) | 30 | 50 |
| Layer 3 (Industrial Agent) | 20 | 40 |
| 評価サブセット | `evaluation_set_v1.yaml` (15問) | `evaluation_set_v2.yaml` (30問) を追加 |
| Judge | v1 | **Judge v2** を追加 |
| スキーマ | v1.0 | v1.0 と **後方互換** の v1.1 拡張 |

v1.0 の 90 問および `evaluation_set_v1.yaml` は **凍結したまま変更していない**。v1.1 は追加・拡張のみで構成されている。

### 2.2 新カテゴリ追加

| Layer | カテゴリ | ID prefix | 追加問題数 |
|---|---|---|---:|
| 1 | Maintenance & Engineering | `IK-MAINT-` | 10 |
| 2 | Risk Tradeoff | `IR-RT-` | 10 |
| 2 | Data Integrity | `IR-DI-` | 10 |
| 3 | Agent Safety | `IA-AS-` | 5 |
| 3 | Structured Decision | `IA-SD-` | 5 |
| 3 | Tool Trajectory | `IA-TT-` | 5 |
| 3 | Human-in-the-Loop Boundary | `IA-HILB-` | 5 |

これらは v1.1-pre の段階で `benchmark_data/` 配下に正式追加された。

v1.1 設計時に Change Control / Compliance / Factory Management / Numeric Capacity Planning の追加も計画したが、本プレリリースには未収録であり、v1.1 正式版以降で順次追加する。

### 2.3 スキーマ拡張 (v1.0 後方互換)

v1.1 で追加された任意フィールド:

- `schema_version`
- `score_cap_rules` — 重大欠落時の final_score 上限
- `numeric_checks` — 必須数値の expected_value / tolerance / unit
- `generic_answer_penalty` — 一般論回答の検出条件
- `structured_output_requirements` — 必須フィールドと形式
- `disallowed_answers` — 明示的禁止回答

v1.0 既存 90 問はこれらのフィールドを持たず、Judge v2 はフィールド欠如時に後方互換的に動作する。

### 2.4 Judge v2 導入

`judge_template_v2.md` を新規追加した。主な強化点:

1. **採点優先順位の固定**
   `critical_failures → disallowed_answers → must_have → score_cap_rules → numeric_checks → structured_output_requirements → generic_answer_penalty → nice_to_have → reasoning_quality` の順で適用する。

2. **score cap の明示適用**
   `must_have` 欠落数 (1→max4 / 2→max3 / 3↑→max2)、`numeric_checks` 失敗率 20% 以上 (max3)、`generic_answer_penalty` (max2)、`structured_output_requirements` 欠落 (2 個以上→max3 / 4 個以上→max2) を機械的に適用する。

3. **Evidence requirement**
   加点項目は回答本文からの引用または要約を必須とし、根拠なしの加点を禁止する。

4. **Output format**
   採点結果は JSON で出力し、`critical_failure_triggered` / `must_have_results` / `numeric_check_results` / `score_cap_rule_results` / `applicable_score_caps` / `final_score` を含める。

### 2.5 evaluation_set_v2 導入

`evaluation_set_v2.yaml` (30 問) を新規追加した。

- Knowledge / Reasoning / Agent 各 10 問
- v1.1 追加カテゴリ (Maintenance & Engineering / Risk Tradeoff / Data Integrity / Agent Safety / Tool Trajectory / HIL Boundary / Structured Decision) を重点的に含む
- v1.0 既存カテゴリも一部含め、Judge v2 の後方互換性を検証可能にした
- difficulty 3〜5、domain も automotive / electronics / general_manufacturing / heavy_machinery / medical_device に分散

### 2.6 識別力改善

匿名化モデル 4 つ × 30 問 = 120 件の Judge v2 全件採点を実施した結果:

| 指標 | v1.0 課題 | v1.1-pre 結果 |
|---|---|---|
| スコア分布 | 4〜5 点へ集中 | 1〜5 点に分布 |
| モデル間平均差 | 小さい (識別力不足) | **2.67 点** |
| score_cap 発動率 | 該当機能なし | 65.0% |
| numeric_check 失敗率平均 | 該当機能なし | 0.333 |
| generic_penalty 発動率 | 該当機能なし | 25.8% |
| critical_failure 発動件数 | ほぼ発動せず | 6 件 / 120 件 |

最も識別力が高かったカテゴリ:

| category | avg_final_score |
|---|---:|
| hil_boundary | 2.00 |
| agent_safety | 2.63 |
| tool_trajectory | 2.63 |
| data_integrity | 2.70 |

Agent 層が最も難しく、HIL Boundary、Tool Trajectory、Agent Safety がモデル差検出に強く効いた。

## 4. 非公開対象 (本リリースに含めないもの)

以下はリポジトリに公開しない:

- モデル ID 対応表
- 提供元名・製品名・運用名
- API キー、シークレット、認証情報

これらは `.gitignore` または運用ルールにより公開リポジトリへ含まないものとする。
