# Industrial Agent Benchmark v1.1.0-pre Release Notes

v1.1.0-pre is a public prerelease of Industrial Agent Benchmark. It expands the dataset and adds Judge v2 evaluation materials while keeping generated answer/result artifacts out of the public repository.

## Summary

- 140 public benchmark problem files.
- `evaluation_set_v2.yaml` with 30 selected questions.
- Judge v2 template with score caps, numeric checks, generic-answer penalties, structured-output requirements, disallowed answers, and critical failures.
- Public-safe README Quick Start.

## Public Contents

- `benchmark_data/`
- `benchmark_data/index.yaml`
- `benchmark_data/index.csv`
- `evaluation_set_v1.yaml`
- `evaluation_set_v2.yaml`
- `judge_template.md`
- `judge_template_v2.md`
- `scripts/generate_dataset.py`
- `scripts/validate_dataset.py`
- `scripts/prepare_judge_inputs_v2.py`
- public documentation under `docs/`

## Excluded From Public Release

- `results_v2/`
- `results/`
- `datasets/results/`
- raw model answers
- generated judge inputs
- judge outputs
- model-specific evaluation results
- unpublished reports
- provider/model-name mappings
- API keys, `.env` files, tokens, and credentials

## Fresh-Clone Quick Start

```bash
pip install pyyaml
python scripts/validate_dataset.py
```

Expected validation result:

```text
Checked: 140 problem files
Errors: 0
Warnings:0
```

## Judge v2 Workflow

Judge v2 input generation requires locally generated answer files:

```text
results_v2/<model_id>/answers/<question_id>.txt
```

`results_v2/` is intentionally excluded from the public repository. After generating local answers, prepare judge inputs with:

```bash
python scripts/prepare_judge_inputs_v2.py <model_id>
```

Example:

```bash
python scripts/prepare_judge_inputs_v2.py my_model
```
