# Industrial Agent Benchmark v1.1 — Design Package Index

本ドキュメントは Industrial Agent Benchmark v1.1 の **設計パッケージのインデックス** であり、
設計レビュー時の入口・読み順・整合性確認結果をまとめる。

v1.1 のスコープと方針は以下 4 ドキュメントに分割されている。本書はそれらを束ねる目次・サマリであり、
新規ルールは定義しない (すべて参照のみ)。

## 4. 設計レビュー時の確認観点 (チェックリスト)

レビュアーへ。以下を順に確認することを推奨する。

### 4.1 目的整合 (design_plan §1, §2)
- [ ] G1–G5 の到達目標が v1.0 課題 R1–R6 / Q1–Q5 / J1–J3 に対応しているか
- [ ] 「単に問題数を増やすのではない」というユーザー方針が、Section 5 の識別力 5 軸として具体化されているか

### 4.2 スキーマ後方互換 (design_plan §4, rubric_design §1)
- [ ] 新規 5 ブロックが既存 v1.0 スキーマと衝突せず、`schema_version` で識別される設計か
- [ ] v1.0 既存 90 問が「対象外」と明示されているか
- [ ] `final_score = max( min(matrix_score, min(caps)) + penalty , floor )` の統一式に齟齬がないか

### 4.3 採点識別力 (rubric_design §2–§7)
- [ ] `score_cap_rules` 9 標準ルールが各識別軸を独立に減点に寄与させられるか
- [ ] `generic_answer_penalty` の 5 条件が Judge LLM で再現性をもって判定できるか
- [ ] `critical_failures` の 7 type 分類カバレッジ目標 (safety≥8, regulatory≥10, …) が新規 90 問の配分と整合しているか

### 4.4 カテゴリ設計 (category_plan §1–§3)
- [ ] 各カテゴリで「差分が出るポイント」が抽象論ではなく **scenario-specific な定量条件 or 禁則** で書かれているか
- [ ] カテゴリ横断カバレッジ表 (§4) で各識別軸が複数カテゴリにまたがって測れる構成になっているか

### 4.5 90 問タイトル (question_titles)
- [ ] 各タイトルが「用語説明 / 一般チェックリスト / 整理せよ」型ではなく、**判断 / 計算 / 構造化出力** を要求する形になっているか
- [ ] 各タイトル末尾の「差分が出る理由」に `num` / `cap` / `gen` / `str` / `dis` / `defer` / `tool` / `hil` のいずれかが含まれているか
- [ ] 既存 v1.0 ID と衝突がないか

### 4.6 触れていないものの確認
- [ ] 既存 v1.0 90 問 YAML、`README.md`、`evaluation_set_v1.yaml`、`results/`、`benchmark_data/index.{yaml,csv}`、`docs/benchmark_spec.md` 本体に変更がないこと
